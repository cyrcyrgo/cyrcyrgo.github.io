# MLTSF 扩展包创建说明书

## 概述

MLTSF (多语言终端模拟器) 支持通过扩展包动态添加新命令和功能。扩展包通常部署在 GitHub 上，程序通过静态 CDN 自动获取并加载。

---

## 一、扩展包目录结构

一个标准的扩展包在 GitHub 仓库中的目录结构如下：

```
你的仓库/
├── extensions/
│   └── 你的扩展包名/
│       ├── extension.json      # [必需] 扩展包配置文件
│       ├── main.js             # [必需] 主脚本文件（注册命令和功能）
│       └── README.md           # [可选] 扩展包说明文档
```

### 命名规范
- **扩展包名**: 小写英文字母 + 连字符，如 `my-tools`、`text-utils`
- **仓库结构**: 所有扩展包统一放在仓库根目录的 `extensions/` 文件夹下

---

## 二、extension.json 配置文件

`extension.json` 是扩展包的元数据文件，定义扩展包的基本信息和提供的命令列表。

### 完整示例

```json
{
    "name": "my-tools",
    "version": "1.0.0",
    "description": "我的工具扩展包，提供常用开发辅助命令",
    "author": "YourName",
    "main": "main.js",
    "commands": [
        {
            "name": "greet",
            "description": "向用户打招呼"
        },
        {
            "name": "calc",
            "description": "简单计算器，支持加减乘除"
        },
        {
            "name": "weather",
            "description": "显示当前天气信息"
        }
    ]
}
```

### 字段说明

| 字段 | 类型 | 必需 | 说明 |
|------|------|------|------|
| `name` | string | 是 | 扩展包名称，应与文件夹名一致 |
| `version` | string | 是 | 版本号，建议遵循 semver 规范 |
| `description` | string | 是 | 扩展包功能描述 |
| `author` | string | 否 | 作者名 |
| `main` | string | 否 | 主脚本文件名，默认 `main.js` |
| `commands` | array | 是 | 提供的命令列表，每个命令包含 `name` 和 `description` |

## 三、main.js 主脚本

`main.js` 是扩展包的核心文件，使用 `window.ExtensionAPI` 注册命令和功能。

### 基础模板

```javascript
// my-tools/main.js - 扩展包主脚本

(function() {
    'use strict';

    // 注册 /greet 命令
    ExtensionAPI.registerCommand('greet', {
        description: '向用户打招呼',
        handler: function(args, terminal) {
            const name = args.join(' ') || '朋友';
            terminal.println(`你好，${name}！欢迎使用 MLTSF 扩展包！`, 'success-line');
            terminal.println(`当前时间: ${new Date().toLocaleString()}`, 'info-line');
        }
    });

    // 注册 /calc 命令
    ExtensionAPI.registerCommand('calc', {
        description: '简单计算器，用法: calc 1 + 2',
        handler: function(args, terminal) {
            if (args.length < 3) {
                terminal.println('用法: calc <数字1> <运算符> <数字2>', 'info-line');
                terminal.println('示例: calc 10 + 5', 'info-line');
                terminal.println('支持的运算符: +, -, *, /', 'info-line');
                return;
            }

            const num1 = parseFloat(args[0]);
            const op = args[1];
            const num2 = parseFloat(args[2]);

            if (isNaN(num1) || isNaN(num2)) {
                terminal.println('错误: 请输入有效的数字', 'error-line');
                return;
            }

            let result;
            switch (op) {
                case '+': result = num1 + num2; break;
                case '-': result = num1 - num2; break;
                case '*': result = num1 * num2; break;
                case '/':
                    if (num2 === 0) {
                        terminal.println('错误: 除数不能为零', 'error-line');
                        return;
                    }
                    result = num1 / num2;
                    break;
                default:
                    terminal.println(`错误: 不支持的运算符 '${op}'`, 'error-line');
                    return;
            }

            terminal.println(`结果: ${num1} ${op} ${num2} = ${result}`, 'success-line');
        }
    });

    // 注册 /weather 命令 - 使用 API 获取天气
    ExtensionAPI.registerCommand('weather', {
        description: '显示当前天气信息（演示功能）',
        handler: function(args, terminal) {
            terminal.println('正在获取天气信息...', 'info-line');

            // 这里可以调用天气API
            // 示例使用模拟数据
            const weatherData = {
                city: args.join(' ') || '北京',
                temperature: Math.floor(Math.random() * 35) + 5,
                condition: ['晴天', '多云', '阴天', '小雨'][Math.floor(Math.random() * 4)],
                humidity: Math.floor(Math.random() * 40) + 40
            };

            terminal.println(`城市: ${weatherData.city}`, 'info-line');
            terminal.println(`温度: ${weatherData.temperature}°C`, 'info-line');
            terminal.println(`天气: ${weatherData.condition}`, 'info-line');
            terminal.println(`湿度: ${weatherData.humidity}%`, 'info-line');
        }
    });

    console.log('[my-tools] 扩展包加载完成');
})();
```

### ExtensionAPI 详细说明

#### 1. `ExtensionAPI.registerCommand(name, definition)`

注册一个新命令。

**参数:**
- `name` (string): 命令名，可以带 `/` 前缀或不带（如 `"greet"` 或 `"/greet"`）
- `definition` (object): 命令定义
  - `description` (string): 命令描述
  - `handler` (function): 命令处理函数 `(args, terminal) => {}`
    - `args` (string[]): 命令参数数组
    - `terminal` (object): 终端对象，可用于输出

#### 2. `ExtensionAPI.print(text, className)`

在终端输出文本。

**参数:**
- `text` (string): 输出文本
- `className` (string, 可选): CSS 类名，用于样式

可用样式类:
- `''` (默认): 默认绿色
- `'error-line'`: 红色错误
- `'success-line'`: 绿色成功
- `'info-line'`: 青色信息
- `'warning-line'`: 黄色警告

#### 3. `ExtensionAPI.getTerminal()`

获取终端对象，可用于调用终端的各种方法。

#### 4. `ExtensionAPI.getProjects()`

获取项目列表对象。

#### 5. `ExtensionAPI.getCurrentEnv()`

获取当前环境名称（如 `'python'`, `'c++'`, `'cmd'`）。

## 四、完整的扩展包示例

### 示例扩展包：text-utils（文本工具）

#### 目录结构
```
extensions/text-utils/
├── extension.json
└── main.js
```

#### extension.json
```json
{
    "name": "text-utils",
    "version": "1.0.0",
    "description": "文本处理工具集",
    "author": "MLTSF",
    "main": "main.js",
    "commands": [
        { "name": "upper", "description": "将文本转为大写" },
        { "name": "lower", "description": "将文本转为小写" },
        { "name": "reverse", "description": "反转文本" },
        { "name": "count", "description": "统计文本字符数" }
    ]
}
```

#### main.js
```javascript
(function() {
    'use strict';

    ExtensionAPI.registerCommand('upper', {
        description: '将文本转为大写',
        handler: function(args, terminal) {
            const text = args.join(' ');
            if (!text) {
                terminal.println('用法: upper <文本>', 'info-line');
                return;
            }
            terminal.println(text.toUpperCase(), 'success-line');
        }
    });

    ExtensionAPI.registerCommand('lower', {
        description: '将文本转为小写',
        handler: function(args, terminal) {
            const text = args.join(' ');
            if (!text) {
                terminal.println('用法: lower <文本>', 'info-line');
                return;
            }
            terminal.println(text.toLowerCase(), 'success-line');
        }
    });

    ExtensionAPI.registerCommand('reverse', {
        description: '反转文本',
        handler: function(args, terminal) {
            const text = args.join(' ');
            if (!text) {
                terminal.println('用法: reverse <文本>', 'info-line');
                return;
            }
            terminal.println(text.split('').reverse().join(''), 'success-line');
        }
    });

    ExtensionAPI.registerCommand('count', {
        description: '统计文本字符数',
        handler: function(args, terminal) {
            const text = args.join(' ');
            if (!text) {
                terminal.println('用法: count <文本>', 'info-line');
                return;
            }
            terminal.println(`字符数: ${text.length}`, 'info-line');
            terminal.println(`单词数: ${text.split(/\s+/).filter(w => w).length}`, 'info-line');
        }
    });
})();
```

## 五、部署到 GitHub

### 步骤

1. **创建 GitHub 仓库**
   ```bash
   git init my-extensions
   cd my-extensions
   mkdir -p extensions/my-tools
   ```

2. **添加扩展包文件**
   ```
   my-extensions/
   └── extensions/
       └── my-tools/
           ├── extension.json
           └── main.js
   ```

3. **提交并推送到 GitHub**
   ```bash
   git add .
   git commit -m "添加 my-tools 扩展包"
   git remote add origin https://github.com/你的用户名/你的仓库.git
   git push -u origin main
   ```

4. **在 MLTSF 中加载**
   在终端输入：
   ```
   /ap my-tools 你的用户名 你的仓库
   ```

## 六、命令冲突处理

当扩展包的命令与内置命令或已加载的扩展命令冲突时，MLTSF 会提示用户选择：

```
命令 'greet' 与内置命令冲突
命令 'calc' 与已加载扩展包 other-tools 冲突

请选择处理方式:
  1. 保留旧的（跳过冲突命令，加载其余部分）
  2. 覆盖旧的（使用扩展包的新命令）
  3. 取消加载该扩展包
冲突解决> _
```

用户输入 `1`、`2` 或 `3` 进行选择。

## 七、最佳实践

1. **模块化**: 每个命令用 IIFE 包裹，避免变量污染
2. **错误处理**: 对用户输入做验证，提供友好的错误提示
3. **帮助信息**: 命令在参数不足时显示用法说明
4. **命名规范**: 命令名使用小写英文，用连字符分隔（如 `text-utils`）
5. **版本管理**: 更新扩展包时递增版本号
6. **测试**: 在本地先用简单功能测试，确认无误后上传

## 九、命令管理

| MLTSF 命令 | 功能 |
|-----------|------|
| `/ap <包名>` | 从注册表加载扩展包 |
| `/ap <包名> <用户> <仓库>` | 从GitHub加载并注册扩展包 |
| `/rm <包名>` | 卸载扩展包 |
| `/lxe` | 列出已加载的扩展包 |
| `/查看扩展包` | 查看注册表中的可用扩展包 |
