Bug Fix Verification

Bug 1: Race Condition - Missing isRunning check in runSelection()

Location: py/js/app.js - runSelection() method

Description: When code is already running, pressing Shift+F5 (Run Selection) would start a second concurrent execution. Both executions share the same outputBuffer/errorBuffer in pyodide-manager, causing output corruption.

Trigger Scenario:

  1. Run a long-running Python script (e.g., import time; time.sleep(10))
  2. While it's running, select some code and press Shift+F5 to run selection
  3. Before fix: Two concurrent executions, output interleaved/corrupted
  4. After fix: Toast message "Code is already running, please wait..." appears, no concurrent execution

✓ Expected Behavior After Fix:

  • If this.isRunning === true, runSelection() rejects immediately
  • Shows toast warning to user
  • No concurrent execution → no output buffer corruption
Bug 2: Silent data loss - stderr output silently dropped in runSelection()

Location: py/js/app.js - runSelection() method

Description: _executeCode() correctly handles result.stderr and displays it to the console. But runSelection() completely ignored result.stderr → any warning/error output from Python is silently lost.

Trigger Scenario:

  1. Select code that produces stderr output (e.g., import warnings; warnings.warn("test warning"))
  2. Run selection with Shift+F5
  3. Before fix: The warning never appears in console → user doesn't see it
  4. After fix: The warning appears in console as error output

✓ Expected Behavior After Fix:

  • If result.stderr exists, it's written to console with 'error' type
  • No silent data loss → user sees all warnings/errors
✓ Fix Applied Summary
  • Bug 1 (Race Condition): Added if (this.isRunning) check before starting execution
  • Bug 2 (Silent data loss): Added if (result.stderr) output handling, consistent with _executeCode()
  • Additional: Added missing try-catch for exceptions in runSelection(), consistent with _executeCode() → prevents hanging isRunning = true
  • Added: Translation keys for toast.alreadyRunning in both Chinese and English

Fix is minimal, focused, and doesn't change any other behavior.
